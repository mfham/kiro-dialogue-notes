# レポート生成スキル

## 役割
指定期間の学びを集約し、成長サマリーを生成する。

## 月次レポート生成フロー

### 1. 対象期間の学びファイルを収集
```bash
CURRENT_MONTH=$(date +%Y-%m)
FILES=(~/.kiro/learnings/${CURRENT_MONTH}-*.md)
```

### 2. カテゴリ別集計
```bash
grep -h "category:" "${FILES[@]}" | \
  sed 's/category: \[//; s/\]//; s/, /\n/g' | \
  sort | uniq -c | sort -rn
```

### 3. タグランキング
```bash
grep -h "tags:" "${FILES[@]}" | \
  sed 's/tags: \[//; s/\]//; s/, /\n/g' | \
  sort | uniq -c | sort -rn | head -10
```

## レポート形式

```markdown
# 📊 学びレポート: 2026年2月

## サマリー
- **期間:** 2026年2月1日 〜 2026年2月28日
- **学び総数:** 15件
- **記録日数:** 12日
- **平均:** 1.25件/日

## 📈 カテゴリ別分析

| カテゴリ | 件数 | 割合 |
|---------|------|------|
| Code Quality | 5 | 33% |
| Security | 4 | 27% |
| Performance | 3 | 20% |
| Git | 2 | 13% |
| Testing | 1 | 7% |

## 🏷️ 頻出タグ TOP5

1. **review** (8件)
2. **refactoring** (6件)
3. **git** (5件)
4. **typescript** (4件)
5. **testing** (3件)

## ✨ 代表的な学び

### Code Quality
**2026-02-21 | 入力値バリデーションの重要性**
> 入力値のバリデーションが不十分だった。今後は必ず入力値チェックを実装する。

### Security
**2026-02-18 | 認証トークンの安全な管理**
> 環境変数を使ってトークンを管理する方法を学んだ。

## 💡 成長の総括

今月は **Code Quality** と **Security** に特に注力していますね！
レビューを通じた学びが多く、チーム開発での成長が見られます。
来月は **Testing** や **Architecture** にも挑戦してみると、さらにスキルの幅が広がりそうです。

**素晴らしい成長です！この調子で続けましょう！🚀**
```

## 年次レポート

年次レポートは月次レポートを拡張：
- 月別の学び数推移グラフ
- 年間で最も成長したカテゴリ
- 年間総学び数とマイルストーン
- 来年の目標提案

## 使用例

- 「今月の学びをまとめて」
- 「2026年1月のレポートを生成」
- 「今年の成長サマリーを見せて」
